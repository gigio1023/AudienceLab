# 🏆 해커톤 우승 전략: Scalable Cognitive Simulation

> **Vision**: **대규모 인지 지능(Crowd)**과 **정밀한 행동 제어(Hero)**를 결합하여, **가장 현실적이고 경제적인 마케팅 성과 예측 시뮬레이터**를 구현한다.
> **Goal**: 6시간 내에 **기술력(Impact)**과 **실현 가능성(Reality)**을 모두 보여주며 **해커톤 1등**을 달성한다.

## 1. The Challenge (왜 기존 방식은 실패하는가)
마케팅 성과 예측("이 광고가 성공할까?")을 위해서는 해커톤 환경에서 양립하기 어려운 두 가지 조건이 필요합니다.
1.  **규모 (Scale, N=100+)**: 통계적 유의성을 위해 다양한 페르소나의 데이터가 필요합니다. 하지만 로컬에서 크롬 브라우저 100개를 띄우면 노트북이 멈춥니다.
2.  **깊이 (Depth, Computer Use)**: 최신 AI 기술(Vision, Mouse Control)을 보여주고 싶지만, 이는 느리고 리소스가 많이 듭니다.

**함정(Trap)**: "50개의 에이전트를 모두 브라우저로 띄우자!" -> 데모 시연 때 렉 걸리고 망함.
**해법(Pivot)**: **"Brain(두뇌)"과 "Body(신체)"를 분리한 하이브리드 아키텍처.**

## 2. Core Architecture: "The Hero & The Crowd"

시뮬레이션 계층을 **인지적 의사결정(The Crowd)**과 **행동적 검증(The Hero)**으로 명확히 나눕니다.

### 🎭 Component A: "The Crowd" (대규모 인지 스웜)
*   **정의**: 99+개의 Headless Agent가 순수 VLM + API 기반으로 동작.
*   **역할**: **"Brain 시뮬레이션" (마케팅 성과 예측 담당)**.
*   **로직**:
    *   **입력**: 캠페인 이미지 + 캡션 + 페르소나 프로필(예: "30대 비건 워킹맘").
    *   **인지(LLM)**: "이 가죽 가방 광고, 내 신념에 반하는데? 불쾌하다." (내적 독백).
    *   **결정**: "부정적인 댓글을 달아야겠다."
    *   **행동**: API를 통해 결과만 전송 (브라우저 렌더링 비용 0).
*   **효과**: 브라우저 없이 **100배 빠른 속도**로 **통계적 유의성(N=100)**을 확보하고, 마케팅 성과(반응률)를 정확히 예측합니다.

### 🦸 Component B: "The Hero" (시각적 Computer Use 에이전트)
*   **정의**: Playwright + VLM 기반의 풀 브라우저(Full Browser) 에이전트, 단 **1개**.
*   **역할**: **"Body 시뮬레이션" (UX 및 기술력 검증 담당)**.
*   **로직**:
    *   **시각(Vision)**: 'The Crowd'가 와글와글 댓글을 달아놓은 피드를 스크린샷으로 봅니다.
    *   **제어(Control)**: "좋아요 버튼이 여기 있네" -> 좌표 계산.
    *   **행동(Action)**: 실제로 마우스 커서를 움직여 클릭합니다.
*   **효과**: **"우리 팀은 Computer Use 기술을 완벽히 구현할 수 있다"**는 것을 시각적으로 증명합니다. (Showcase)

## 3. Execution Plan (6시간 타임라인)

| 단계 | 시간 | Engine Team (Backend/AI) | Agent Team (Frontend/Visual) |
|---|---|---|---|
| **P1: Ground Truth** | 11:30 - 12:00 | **수동 데이터 수집**: 인플루언서 3명, 포스트 15개, 댓글 50개를 직접 긁어서 `seeds.json` 작성. (가장 중요) | **환경 셋업**: Pixelfed Docker 구동, 로컬 접속 확인. |
| **P2: Foundation** | 12:00 - 13:30 | **데이터 주입**: `seeds.json`을 Pixelfed DB에 넣어 "살아있는 SNS" 상태 만들기. | **Visual 기초**: Playwright 루프 구현 (로그인 -> 스크롤 -> 스크린샷). |
| **P3: The Swarm** | 13:30 - 15:30 | **Ghost Swarm**: `ThreadPool`로 VLM 병렬 호출 -> API 폭격. **댓글 내용의 퀄리티(페르소나 반영)**에 집중. | **Visual 튜닝**: "읽고 -> 멈칫하고 -> 클릭하는" 사람 같은 타이밍(Human-like Timing) 튜닝. |
| **P4: Integration** | 15:30 - 16:30 | **대시보드 연동**: 실시간 API 로그를 대시보드에 스트리밍 ("Agent #42님이 댓글을 남겼습니다"). | **상호작용 확인**: 'Hero'가 'Crowd'가 남긴 댓글을 보고 반응하는 시나리오 테스트. |
| **P5: Showtime** | 16:30 - 18:30 | **시나리오 리허설**: 데모 영상 녹화 및 편집. | **마무리**: UI 폴리싱, README 및 발표 자료 작성. |

## 4. Key Differentiators (우승 포인트)

1.  **진짜 같은 데이터 (Realism)**: 랜덤 데이터가 아니라, 실제 인스타그램에서 **수동 채굴한 데이터(Ground Truth)**를 씨앗(Seed)으로 사용하여 현실성을 극대화합니다.
2.  **타당한 예측 논리 (Logic)**: 단순 버튼 클릭 봇이 아닙니다. 100명의 페르소나가 콘텐츠를 보고 **사고하는 과정(Chain-of-Thought)**을 시뮬레이션하여 마케팅 성과를 예측합니다.
3.  **경제성 & 확장성 (Viability)**: 100개를 다 브라우저로 돌리는 비효율($9.50/회) 대신, 하이브리드 구조($0.65/회)를 통해 **93% 비용 절감**을 달성, 실제 B2B 사업화 가능성을 증명합니다.
